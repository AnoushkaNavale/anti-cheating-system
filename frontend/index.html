⬡ HOW THE SYSTEM WORKS
STEP 1 — Start Exam
Enter Student ID and Exam ID → click START EXAM
All student actions are now being monitored in real time
STEP 2 — Events Are Captured
Tab switches, copy/paste, right clicks, and idle time
are all logged and sent to the backend server
STEP 3 — Finalize & Store
Click FINALIZE & STORE when exam ends
Backend batches all events → generates SHA-256 hash
Uploads JSON file to IPFS → receives CID
STEP 4 — Blockchain Storage
CID and hash are stored permanently on the blockchain
via the ExamLogger smart contract
This record can never be changed or deleted
STEP 5 — Verify Integrity
Go to VERIFY tab → paste CID and hash
System fetches file from IPFS → recomputes hash
Compares with blockchain record → shows VALID or TAMPERED
